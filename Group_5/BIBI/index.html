  • (C language)
    ແນະນໍາພາສາ C

    ໃນບົດນີ້ຈະເປັນການແນະນໍາພາບລວມຂອງພາສາ C ເພື່ອໃຫ້ຜູ້ອ່ານໄດ້ຮູ້ແນວຄິດພື້ນຖານ ແລະ ເບິ່ງພາບລວມຂອງພາສາ C ໄດ້ໂດຍຫຍໍ້ ກ່ອນຈະສຶກສາລາຍລະອຽດໃນບົດຕໍ່ໆໄປ.

  • 1.1 ປະຫວັດຄວາມເປັນມາຂອງພາສາ C


    • ປີ ຄ.ສ 1972 Dennis Ritchie ເປັນຜູ້ຄິດຄົ້ນສ້າງພາສາ C ຂຶ້ນເປັນຄັ້ງທໍາອິດໂດຍ ພັດທະນາຈາກພາສາ B ແລະ ພາສາ BCPL ແຕ່ໃນຕອນນັ້ນຍັງບໍ່ໄດ້ມີການນຳໃຊ້ຢ່າງກວ້າງຂວາງ ຈົນ ມາຮອດປີ 1978 Brian Kemighan ໄດ້ຮ່ວມກັບ Dennis Ritchie ເພື່ອພັດທະນາຖານພາສາ C ຂຶ້ນມາ ເອີ້ນວ່າ K&R (Kemighan& Ritchie) ແລະ ໄດ້ຂຽນໜັງສືຊື່ The C programming Language ອອກ ມາເປັນເຫຼັ້ມທໍາອິດ ແລະ ໄດ້ເຮັດໃຫ້ເລີ່ມມີຜູ້ສົນໃຈພາສາ C ຫຼາຍຂຶ້ນ, ດ້ວຍການຂະຫຍາຍຕົວຂອງ ພາສາ C ທີ່ສາມາດປັບໃຊ້ງານກັບຄອມພິວເຕີຊະນິດຕ່າງໆໄດ້ ເຮັດໃຫ້ພາສາ C ໄດ້ຮັບຄວາມນິຍົມເພີ່ມ ຫຼາຍຂຶ້ນເລື້ອຍໆຈົນແພ່ອອກໄປທົ່ວໂລກຈົນມີບໍລິສັດຕ່າງໆສ້າງ ແລະ ຜະລິດພາສາ C ອອກມາເປັນ ນວນຫຼວງຫຼາຍ, ເກີດມີພາສາ C ຫຼາກຫຼາຍຮູບແບບເນື່ອງຈາກຂະນະນັ້ນຍັງບໍ່ມີການກຳນົດມາດຕະຖານ ສຳລັບການສ້າງພາສາ C. ດັ່ງນັ້ນ, ປີ 1988 Ritchie ແລະ Kemighan ຈິ່ງໄດ້ຮ່ວມກັບ ANSI (American National Standards Institute) ສ້າງມາດຕະຖານພາສາ C ຂຶ້ນມາເອີ້ນວ່າ: ANSI C ເພື່ອ ໃຫ້ເປັນຕົວກຳນົດມາດຕະຖານໃນການສ້າງພາສາ C ໃຫ້ລຸ້ນຕໍ່ໆໄປ.
      ໃນປັດຈະບັນ, ພາສາ C ຍັງໄດ້ຮັບຄວາມນິຍົມ ແລະ ໃຊ້ງານຢ່າງກວ້າງຂວາງ ເນື່ອງຈາກເປັນ ພາສາລະດັບກາງ (middle-level language) ທີ່ເໝາະກັບການຂຽນໂປຣແກຣມແບບໂຄງສ້າງ ແລະ ເປັນ ພາສາທີ່ມີການຂະຫຍາຍຕົວຫຼາຍ ຄື ສາມາດໃຊ້ງານກັບເຄື່ອງຕ່າງໆໄດ້ ແລະ ສໍາຄັນໃນປັດຈຸບັນພາສາ ໂປຣແກຣມລຸ້ນໃໝ່ເຊັ່ນ: C++, Perl, Java, C# ແລະ ອື່ນໆ ກໍ່ຍັງໃຊ້ຫຼັກການຂອງພາສາ C ເປັນ ພື້ນຖານ, ກ່າວຄືຫາກມີພື້ນຖານຂອງພາສາ C ກ່ອນກໍ່ສາມາດສຶກສາພາສາລຸ້ນໃໝ່ນີ້ໄດ້ງ່າຍຂຶ້ນ.

  • 1.2 ຂັ້ນຕອນໃນການພັດທະນາໂປຣແກຣມພາສາ C

    1.2.1 ຂັ້ນຕອນທີ 1 : ການຂຽນໂປຣແກມ (Source Code)

    ໃຊ້ editor ຂຽນໂປຣແກຣມພາສາ C ແລະ ເຮັດການບັນທຶກໄຟລ໌ໃຫ້ມີນາມສະກຸນເປັນ C ເຊັ່ນ: test.c ເປັນຕົ້ນ.


    ໝາຍເຫດ:

    editor ເປັນໂປຣແກຣມທີ່ໃຊ້ສຳລັບຂຽນໂປຣແກຣມ ໂດຍຕົວຢ່າງຂອງ editor ທີ່ນິຍົມນໍາ ມາໃຊ້ຂຽນໂປຣແກຣມໄດ້ແກ່ Notepad, edit ຂອງ DOS, Textpad(www.textpad.com) Editplus (www.editplus.com) • ເປັນຕົ້ນ. ຜູ້ຂຽນໂປຣແກຣມສາມາດເລືອກໃຊ້ໂປຣແກຣມໃດກໍ່ໄດ້ ແລ້ວແຕ່ຄວາມມັກຂອງແຕ່ລະຄົນ.

  • 1.2.2 ຂັ້ນຕອນທີ 2: ຄອມໄພລໂປຣແກຣມ (Compile)

    ນຳ Source Code ຈາກຂັ້ນຕອນທີ 1 ມາຄອມໄພລ໌ ເພື່ອປ່ຽນຈາກພາສາທີ່ມີມະນຸດເຂົ້າໃຈໄປເປັນ ພາສາຄອມທີ່ຄອມພິວເຕີສາມາດເຂົ້າໃຈໄດ້, ໃນຂັ້ນຕອນຄອມໄພລ໌ເລີ້ຈະເຮັດການກວດສອບ Source Code ວ່າເກີດຂໍ້ຜິດພາດ ຫຼື ບໍ່. • ຫາກເກີດຂໍ້ຜິດພາດຈະແຈ້ງໃຫ້ຜູ້ຂຽນໂປຣແກຣມຮູ້, ຜູ້ຂຽນໂປຣແກຣມຈະຕ້ອງກັບໄປແກ້ໄຂຄືນ ແລະ ເຮັດການຄອມເພລໍໂປຣແກຣມອີກເທື່ອໜຶ່ງ. • ຖ້າບໍ່ເກີດມີຄວາມຜິດພາດ, ຄອມໄພລ໌ຈະແປໄຟລ໌ Source Code ຈາກພາສາ C ໄປເປັນພາສາ ຄອມ (ໄຟລ໌ນາມສະກຸນ .obj) ເຊັ່ນ: ຖ້າໄຟລ໌ SeurceCode ຊື່ text.c ກໍ່ຈະຖືກແປເປັນໄຟລ໌ຊື່ test.obj ເຊິ່ງເກັບພາສາເຄື່ອງໄວ້ເປັນຕົ້ນ.


    ໝາຍເຫດ:

    ຄອມໄພລ໌ເລີ້ (Compiler) ເປັນຕົວປ່ຽນພາສາຮູບແບບໜຶ່ງ, ມີໜ້າທີ່ຫຼັກຄື ປ່ຽນພາສາທີ່ ມະນຸດຂຽນຂຶ້ນເປັນພາສາຂອງຄອມພິວເຕີ້ ໂດຍຄອມໄພເລີ້ຂອງພາສາ C ຄື: C Compiler ເຊິ່ງ ຫຼັກການທີ່ຄອມໄພລ໌ເລີ້ໃຊ້ເອີ້ນ ຄອມໄພລ໌ Compile ໂດຍຈະເຮັດການອ່ານໂປຣແກຣມພາສາ C ທັງ ໝົດຕັ້ງແຕ່ຕົ້ນຈົນຈົບແລ້ວເຮັດການແປຜົນໃນທີ່ສຸດ. ນອກຈາກຄອມໄພລ໌ເລີ້ແລ້ວຍັງມີຕົວປ່ຽນພາສາອີກຮູບແບບໜຶ່ງທີ່ເອີ້ນວ່າ: ອິນເຕີພຣີເຕີ້ (Interpreter) ເຊິ່ງຫຼັກການເຮັດວຽກຈະແຕກຕ່າງຈາກຄອມໄພລ໌ເລີ້ຄື: ອິນເຕີພຣີເຕີ້ຈະອ່ານ ແລະ ແປໂປຣແກຣມໄປ ເທື່ອລະບັນທັດ, ເມື່ອແປຜົນບັນທັດໜຶ່ງສຳເລັດແລ້ວຈະເຮັດວຽກຕາມຄຳສັ່ງໃນບັນທັດນັ້ນແລ້ວຈຶ່ງເຮັດ ການແປຜົນ ແລະ ເຮັດຕາມຄຳສັ່ງບັນທັດຕໍ່ໄປ, ຫຼັກການທີ່ອິນເຕີພຣີເຕີ້ໃຊ້ເອີ້ນວ່າ: ອິນເຕີ້ພຣີ (interpret) ຫຼາຍຂຶ້ນເລື້ອຍໆຈົນແພ່ອອກໄປທົ່ວໂລກຈົນມີບໍລິສັດຕ່າງໆສ້າງ ແລະ ຜະລິດພາສາ C ອອກມາເປັນ ນວນຫຼວງຫຼາຍ, ເກີດມີພາສາ C ຫຼາກຫຼາຍຮູບແບບເນື່ອງຈາກຂະນະນັ້ນຍັງບໍ່ມີການກຳນົດມາດຕະຖານ ສຳລັບການສ້າງພາສາ C.

  • ດັ່ງນັ້ນ, ປີ 1988 Ritchie ແລະ Kemighan

    ຈິ່ງໄດ້ຮ່ວມກັບ ANSI (American National Standards Institute) ສ້າງມາດຕະຖານພາສາ C ຂຶ້ນມາເອີ້ນວ່າ: ANSI C ເພື່ອ ໃຫ້ເປັນຕົວກຳນົດມາດຕະຖານໃນການສ້າງພາສາ C ໃຫ້ລຸ້ນຕໍ່ໆໄປ. ໃນປັດຈະບັນ, ພາສາ C ຍັງໄດ້ຮັບຄວາມນິຍົມ ແລະ ໃຊ້ງານຢ່າງກວ້າງຂວາງ ເນື່ອງຈາກເປັນ ພາສາລະດັບກາງ (middle-level language) ທີ່ເໝາະກັບການຂຽນໂປຣແກຣມແບບໂຄງສ້າງ ແລະ ເປັນ ພາສາທີ່ມີການຂະຫຍາຍຕົວຫຼາຍ ຄື ສາມາດໃຊ້ງານກັບເຄື່ອງຕ່າງໆໄດ້ ແລະ ສໍາຄັນໃນປັດຈຸບັນພາສາ ໂປຣແກຣມລຸ້ນໃໝ່ເຊັ່ນ: C++, Perl, Java, C# ແລະ ອື່ນໆ ກໍ່ຍັງໃຊ້ຫຼັກການຂອງພາສາ C ເປັນ ພື້ນຖານ, ກ່າວຄືຫາກມີພື້ນຖານຂອງພາສາ C ກ່ອນກໍ່ສາມາດສຶກສາພາສາລຸ້ນໃໝ່ນີ້ໄດ້ງ່າຍຂຶ້ນ.

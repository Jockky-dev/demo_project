Introduction of Dart Language Method Main ການສະແດງຜົນຂໍ້ມູນ ການຂຽນຄຳອະທິບາຍ (Comment) ໂຕແປ(Variable) ແລະ ຊະນິດຂໍ້ມູນ(Data Type) ຮູບແບບໂຕແປໃນພາສາ Dart ການນິຍາມໂຕແປ Static Type ການນິຍາມໂຕແປ Dynamic Type ການນິຍາມຄ່າຄົງທີ Constant ການນິຍາມຄ່າຄົງທີ Final ກົດການຕັ້ງຊື່ໂຕແປ ໂຕດຳເນີນການ (Operator) ໂຕດຳເນີນການທາງຄະນິດສາດ ໂຕດຳເນີນການປຽບທຽບແບບ boolean ໂຕດຳເນີນການເພີ່ມຄ່າ-ລົດຄ່າ Compound Assignment ໂຄງສ້າງຄວບຄູມ (Control Structure) ແບບມີເງື່ອນໄຂ (Condition) ຮູບແບບຄຳສັ່ງແບບເງື່ອນໄຂດຽວ ຮູບແບບຄຳສັ່ງແບບ 2 ເງື່ອນໄຂ ຮູບແບບຄຳສັ່ງແບບຫຼາຍເງື່ອນໄຂ ໂຕດຳເນີນການທາງຕັກກະສາດ ແບບທຳຊ້ຳ(Loop) ຄຳສັ່ງ While ຄຳສັ່ງ For Do..While ຄຳສັ່ງທີ່ກ່ຽວຂ້ອງກັບ Loop ຂໍ້ແຕກຕ່າງ ແລະ ການໃຊ້ງານ Loop ຟັ່ງຊັ່ນ (Function) ຮູບແບບຂອງ Function ຂອບເຂດໂຕແປ

ການຂຽນຄຳອະທິບາຍ (Comment)

ວິທີທີ່ 1 ໂດຍໃຊ້ເຄື່ອງໝາຍ Slash (/) ໃຊ້ໃນການອະທິບາຍຄຳສັ່ງສັ້ນໆໃນແຖວດຽວ

ວິທີທີ່ 2 ຂຽນຄຳອະທິບາຍໄວ້ໃນເຄື່ອງໝາຍ /* ... */ ໃຊ້ໃນການອະທິບາຍຄຳສັ່ງຍາວໆຫຼາຍແຖວ

ໂຕແປ(Variable) ແລະ ຊະນິດຂໍ້ມູນ(Data Type)

ໂຕແປ ຫຼື Variable ຄືຊື່ທີ່ຖືກນິຍາມຂຶ້ນມາເພື່ອໃຊ້ເກັບຄ່າຂໍ້ມູນສຳຫຼັບນຳໄປໃຊ້ງານໃນໂປຣແກຣມ ໂດຍຂໍ້ມູນອາດຈະປະກອບໄປດ້ວຍຂໍ້ຄວາມ ຕົວເລກ ຕົວອັກສອນ ຫຼື ຜົນລັບຈາກການປະມວນຜົນຂໍ້ມູນ

 ⚬ ຊະນິດຂໍ້ມູນ:

  1. boolean: ແມ່ນຄ່າທາງຕັກກະສາດເຊັ່ນ: True/False

  2. num:

  - ແມ່ນຕົວເລກທີ່ບໍ່ມີຈຸດທົດສະນິຍົມ(int) ເຊັ່ນ: 10, 20, 30

  - ແມ່ນຕົວເລກຈຸດທົດສະນິຍົມ(double) ເຊັ່ນ: 30,15

  3. string: ຂໍ້ຄວາມ ເຊັ່ນ: Vilaysone

  4. List: ແມ່ນໂຄງສ້າງຂໍ້ມູນ ເຊັ່ນ: ["First", "Second", Third]

  5. Dynamic: ແມ່ນໂຕແປທີ່ປ່ຽນຄ່າໄດ້ ເຊັ່ນ: 20, 30, 15, True, "jockky"

ຮູບແບບໂຕແປໃນພາສາ Dart

▸ໂຕແປແບບ Dynamic Typing ຄືຊະນິດໂຕແປຈະເປັນຫຍັງກໍ່ໄດ້ຕາມຄ່າທີ່ໂຕມັນເກັບໂດຍບໍ່ຕ້ອງປະກາດຊະນິດຂໍ້ມູນ

▸ໂຕແປແບບ Static Typing ຕ້ອງປະກາດຊະນິດຂໍ້ມູນໃນຕອນເລີ່ມຕົ່ນເຊັ່ນ: int, double... ເພື່ອບອກຊະນິດຂໍ້ມູນ

ການນິຍາມໂຕແປ Static Type

ຊະນິດຂໍ້ມູນ ຊື່ໂຕແປ;

ຊະນິດຂໍ້ມູນ ຊື່ໂຕແປ = ຄ່າເລີ່ມຕົ້ນ;

ແບບຫຼາຍໂຕແປໃນແຖວດຽວ

ຊະນິດຂໍ້ມູນ ຊື່ໂຕແປ = ຄ່າເລີ່ມຕົ້ນ, ຊະນິດຂໍ້ມູນ = ຄ່າເລີ່ມຕົ້ນ

ຖ້າບໍ່ໄດ້ກຳໜົດຄ່າ ຈະມີຄ່າເປັນ null ໂດຍອັດຕະໂນມັດ

ການນິຍາມໂຕແປ Dynamic Type

var ຊື່ໂຕແປ;

var ຊື່ໂຕແປ = ຄ່າເລີ່ມຕົ້ນ;

var ຊື່ໂຕແປ = ຄ່າເລີ່ມຕົ້ນ, ຊື່ໂຕແປ = ຄ່າເລີ່ມຕົ້ນ

ການນິຍາມຄ່າຄົງທີ Constant

const ຊະນິດຂໍ້ມູນ ຊື່ໂຕແປ;

const ຊະນິດຂໍ້ມູນ ຊື່ໂຕແປ = ຄ່າເລີ່ມຕົ້ນ;

ການນິຍາມຄ່າຄົງທີ Final

final ຊະນິດຂໍ້ມູນ ຊື່ໂຕແປ;

final ຊະນິດຂໍ້ມູນ ຊື່ໂຕແປ = ຄ່າເລີ່ມຕົ້ນ;

▸ + ໝາຍຄວາມວ່າ ບວກ

▸ - ໝາຍຄວາມວ່າ ລົບ

▸ / ໝາຍຄວາມວ່າ ຫານ

▸ % ໝາຍຄວາມວ່າ ຫານເອົາເສດ

ໂຕດຳເນີນການປຽບທຽບແບບ boolean

▸ == ໝາຍຄວາມວ່າ ເທົ່າກັບ

▸ != ໝາຍຄວາມວ່າ ບໍ່ເທົ່າກັບ

▸ > ໝາຍຄວາມວ່າ ຫຼາຍກວ່າ

▸ < ໝາຍຄວາມວ່າ ນ້ອຍກວ່າ

▸>= ໝາຍຄວາມວ່າ ຫຼາຍກວ່າເທົ່າກັບ

▸<= ໝາຍຄວາມວ່າ ນ້ອຍກວ່າເທົ່າກັບ

>ໂຕດຳເນີນການເພີ່ມຄ່າ-ລົດຄ່າ

▸++ (Prefix) ໂຕຢ່າງ: ++a ໝາຍຄວາມວ່າ ເພີ່ມຄ່າໃຫ້ a ກ່ອນ 1 ຄ່າແລ້ວນຳໄປໃຊ້

▸++ (Postfix) ໂຕຢ່າງ: a++ ໝາຍຄວາມວ່າ ນຳຄ່າປັດຈູບັນໃນ a ໄປໃຊ້ກ່ອນແລ້ວຄ່ອຍເພີ່ມຄ່າ

▸-- (Prefix) ໂຕຢ່າງ: --b ໝາຍຄວາມວ່າ ລົດຄ່າໃຫ້ b ກ່ອນ 1 ຄ່າແລ້ວນຳໄປໃຊ້

▸-- (Postfix) ໂຕຢ່າງ: b-- ໝາຍຄວາມວ່າ ນຳຄ່າປັດຈູບັນໃນ b ໄປໃຊ້ກ່ອນແລ້ວຄ່ອຍລົດຄ່າ

Compound Assignment

▸+= ໂຕຢ່າງ: x+=y ໝາຍຄວາມວ່າ x=x+y

▸-= ໂຕຢ່າງ: x-=y ໝາຍຄວາມວ່າ x=x-y

▸*= ໂຕຢ່າງ: x*=y ໝາຍຄວາມວ່າ x=x*y

▸/= ໂຕຢ່າງ: x~/=y ໝາຍຄວາມວ່າ x=x~/y

▸%= ໂຕຢ່າງ: x%=y ໝາຍຄວາມວ່າ x=x%y

ຄຳສັ່ງ For

• For Loop

ເປັນຮູບແບບທີ່ໃຊ້ໃນການກວດສອບເງື່ອນໄຂ ມີການນຳໜົດຄ່າເລື່ມຕົ້ນ ແລະ ປ່ຽນຄ່າໄປພ້ອມໆກັນ ເມື່ອເງື່ອນໄຂໃນຄຳສັ່ງ for ເປັນຈິງກໍຈະເຮັດວຽກຕາມຄຳສັ່ງທີ່ສະແດງໄວ້ພາຍໃນຄຳສັ່ງ for ໄປເລື້ອຍໆ

ໂຄງສ້າງຄຳສັ່ງ

for(ຄ່າເລີ່ມຕົ້ນຂອງໂຕແປ;ເງື່ອນໄຂ;ປ່ຽນແປງຄ່າໂຕແປ) {
ຄຳສັ່ງເມື່ອເງື່ອນໄຂເປັນຈິງ;
}

Do..While

• Do..While ແມ່ນຈະເຮັດວຽກຕາມຄຳສັ່ງຢ່າງນ້ອຍ 1 ຮອບ ເມື່ອເຮັດວຽກແລ້ວຈະມາກວດສອບເງື່ອນໄຂຄຳສັ່ງ while ຖ້າເງື່ອນໄຂເປັນຈິງຈະວົນກັບຂຶ້ນໄປເຮັດວຽກທີ່ຄຳສັ່ງໃຫມ່ອີກຮອບ ແຕ່ຖ້າບໍ່ເປັນຕຈິງຈະຫຼຸດອອກຈາກ Loop

ໂຄງສ້າງຄຳສັ່ງ


do {

} while(ເງື່ອນໄຂ);


ຄຳສັ່ງທີ່ກ່ຽວຂ້ອງກັບ Loop

• break ຖ້າໂປຣແກຣມພົບຄຳສັ່ງນີ້ຈະຫຼຸດຈາກການເຮັດວຽກໃນ loop ທັນທີເພື່ອໄປເຮັດຄຳສັ່ງອື່ນທີ່ຢູ່ນອກ loop

• continue ຄຳສັ່ງນີ້ຈະເຮັດໃຫ້ຢຸດການເຮັດວຽກແລ້ວຍ້ອນກັບເລີ່ມຕົ້ນການເຮັດວຽກທີ່ຕົ້ນ loop ໃຫມ່

ຂໍ້ແຕກຕ່າງ ແລະ ການໃຊ້ງານ Loop


• For ໃຊ້ໃນກໍລະນີຮູ້ຈຳນວນຮອບທີ່ຊັດເຈນ

• While ໃຊ້ໃນກໍລະນີທີ່ໄມ່ຮູ້ຈຳນວນຮອບ

• Do..While ໃຊ້ໃນກໍລະນີທີ່ຢາກໃຫ້ລອງເຮັດວຽກກ່ອນ 1 ຮອບແລ້ວຈັ່ງເຮັດຊ້ຳໄປເລື້ອຍໆເທົ່າທີ່ເງື່ອນໄຂເປັນຈິງ

ຟັ່ງຊັ່ນ (Function)


ໝາຍເຖິງຊຸດຄຳສັ່ງທີ່ນຳມາຂຽນລວມກັນຢູ່ເປັນກຸ່ມເພື່ອໃຫ້ເອີ້ນໃຊ້ງານຕາມວັດຖຸປະສົງທີ່ຕ້ອງການ ແລະ ລົດຄວາມຊ້ຳຊ້ອນຂອງຄຳສັ່ງທີ່ໃຊ້ຕະຫຼອດ ຟັ່ງຊັ່ນສາມາດໄປໃຊ້ໄດ້ທຸກທີ່ ແລະ ແກ້ໄຂໄດ້ໃນພາຍຫຼັງ ເຮັດໃຫ້ໂຄ້ດໃນໂປຣແກຣມມີລະບຽບ ແລະ ໃຊ້ງານໄດ້ສະດວກຫຼາຍຂຶ້ນ

void ຊື່ຟັ່ງຊັ່ນ() {

}
